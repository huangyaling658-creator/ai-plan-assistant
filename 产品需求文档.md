# AI 计划助手 — 产品需求文档（PRD）

> 版本：v1.0  
> 日期：2026-02-28  
> 状态：MVP 开发阶段

---

## 一、产品概述

### 1.1 产品定位

一款基于 AI 的个人计划管理工具，通过学习用户的历史计划风格，自动生成符合用户个人习惯的日/周/月计划，并持续监督、分析完成情况，形成"越用越懂你"的智能计划闭环。

### 1.2 解决的痛点

| 痛点 | 解决方案 |
|------|---------|
| 每天写计划耗时，或不知道该做什么计划 | AI 根据月目标 + 用户习惯自动生成日/周计划 |
| 没有人监督和分析完成情况 | AI 每日/每周分析完成率，识别拖延模式，给出改进建议 |
| 计划风格不一致，AI 生成的不像"自己写的" | 5 层 Prompt 系统提取并延续用户的个人计划风格 |

### 1.3 产品愿景

- **近期**：单页面 Web 工具，验证"AI 生成计划"核心价值
- **中期**：接入 Supabase 后端，支持用户注册/登录、数据持久化
- **远期**：公开发布，让所有人都能使用

| 角色 | 描述 |

### 3.2 模块一：上传与风格学习

**用户流程**：上传历史计划 → AI 分析 → 生成风格画像

| 功能项 | 详细说明 |
|--------|---------|
| 上传按钮 | 位于左上角，支持图片、文档、纯文字三种输入方式 |
| 历史计划列表 | 展示已上传的所有历史计划，支持查看和删除 |
| 风格分析 | 点击"分析风格"按钮，AI 提取用户计划风格画像（JSON） |
| 风格画像合并 | 每次上传新计划，合并更新画像（不是替换），画像越来越精准 |

**风格画像包含的 15 个维度**：

```json
{
  "format": "时间块/纯列表/分类列表/表格/...",
  "granularity": "粗略/中等/精细",
  "time_preference": "早起型/夜猫型/均衡型",
  "peak_hours": "用户安排重要事项的时段",
  "categories": ["用户常用的分类名称"],
  "priority_markers": "用户标记优先级的方式",
  "recurring_items": ["每天/每周重复出现的固定事项"],
  "task_volume": "每日平均任务数量范围",
  "wording_style": "用户的措辞特点",
  "emoji_usage": "是否使用 emoji 及使用方式",
  "planning_horizon": "计划覆盖的时间跨度偏好",
  "break_pattern": "休息时间安排方式",
  "completion_markers": "用户标记完成状态的方式（如 ✅/❌、删除线、'已完成'文字、勾选框等）",
  "sample_expressions": ["从用户原文提取的5-8个典型表达"],
  "confidence": "high/medium/low — 风格提取的置信度，取决于历史计划的丰富程度"
}
```

**风格兜底机制**：

当用户上传的历史计划过于简单（如只有几个关键词、无明显格式特征），AI 无法提炼出有效风格时，系统需要兜底处理：

- 风格画像的 `confidence` 字段标记为 `"low"`
- 后续计划生成自动采用**默认时间块格式**作为兜底风格：

```
- 08:00-09:00 完成XX任务
- 09:00-10:30 处理XX事项
- 10:30-10:45 休息
- 10:45-12:00 完成XX工作
...
```

- 完成标记兜底为勾选框（☐ 未完成 / ☑ 已完成）
- 随着用户后续的修改和新计划上传，AI 持续学习，`confidence` 逐步提升，兜底风格逐渐被用户真实风格替代

### 3.3 模块二：计划生成

**核心理念**：AI 既要把月计划拆解为周计划、再拆解为日计划，同时也要具备**目标拆解的思考能力**——理解用户的最终目标是什么，反推达成目标需要哪些关键步骤，判断合理的节奏和先后顺序，再结合用户的行为风格（精力分布、执行习惯、擅长领域）来编排具体的执行方案。AI 是用户的**规划参谋**，不仅要模仿风格，更要主动思考"怎样安排才能帮用户真正达成目标"。

**用户流程**：输入月计划 → AI 深度思考目标拆解策略 → 生成周计划 → 生成日计划 → 用户可编辑

| 功能项 | 详细说明 |
|--------|---------|
| 月计划输入 | 支持图片、文档、纯文字三种输入方式（与上传历史计划一致），可随时更新 |
| **AI 目标拆解** | AI 收到月计划后，先进行深度分析：识别核心目标、判断目标间的依赖关系、评估难度和所需时间、规划合理的推进节奏（哪些事先做、哪些并行、哪些需要前置准备），然后再据此生成周/日计划 |
| 周计划生成 | AI 基于目标拆解策略 + 风格画像，生成本周计划（包含本周的重点方向和关键里程碑），可编辑 |
| 日计划生成 | AI 基于周计划 + 风格画像 + 用户精力分布，生成今日计划（高难度任务放在高效时段），可编辑 |
| 完成状态记录 | 完成状态的记录方式由 AI 根据用户历史计划风格决定（如用户习惯用 ✅/❌，则沿用；习惯用删除线，则沿用；习惯用"已完成"文字标注，则沿用），确保与用户自身习惯一致 |
| 计划编辑 | 用户可直接修改 AI 生成的计划内容 |
| 修改追踪 | 系统保存原始版本和修改版本，用于 AI 学习 |

### 3.4 模块三：今日总结

**用户流程**：触发总结 → AI 分析 → 展示报告 → 自动生成明日计划

| 功能项 | 详细说明 |
|--------|---------|
| 完成情况分析 | 分析今日完成率、完成趋势 |
| 未完成项分析 | 识别拖延项，分析原因，给出建议 |
| 用户修改分析 | 检测用户对 AI 计划的修改行为，学习偏好 |
| 改进建议 | 给出具体的效率改进建议 |
| 顺延事项 | 列出需要顺延到明天的未完成项 |

### 3.5 模块四：明日计划生成

| 功能项 | 详细说明 |
|--------|---------|
| 自动预写 | AI 完成今日分析后，自动预写明日计划 |
| 融合策略 | 综合考虑：原目标 + 今日修改偏好 + 未完成项 + 风格画像 |
| 用户确认 | 用户可接受或修改明日计划 |

### 3.6 模块五：设置

| 功能项 | 详细说明 |
|--------|---------|

## 四、AI 系统设计

### 4.1 五层 Prompt 系统

这是产品的核心技术，5 个专职 Prompt 各司其职，形成学习闭环：

```
上传历史计划 ──Prompt 1──→ 风格画像 v1
                                │
月计划输入 ──────Prompt 2──→ 生成周/日计划
                                │
用户修改计划 ──Prompt 3──→ 风格画像 v1.1（更新）
                                │
用户勾选完成 ──Prompt 4──→ 今日总结
                                │
              ──Prompt 5──→ 明日计划（更精准）
                                │
用户再次修改 ──Prompt 3──→ 风格画像 v1.2（再更新）
                                │
                          （持续循环，越用越懂你）
```

#### Prompt 1：风格提取（Style Analysis）

- **触发时机**：用户上传历史计划后
- **输入**：用户上传的历史计划文本/图片内容
- **输出**：结构化 JSON 风格画像
- **关键要求**：分析 15 个维度（含完成标记方式和置信度），输出合法可解析的 JSON；若历史计划过于简单，`confidence` 标记为 `"low"`

#### Prompt 2：计划生成（Plan Generation）

- **触发时机**：用户输入月计划 / 请求生成周/日计划
- **输入**：风格画像 + 月计划 + 历史示例 + 周数
- **输出**：符合用户风格的周计划和日计划
- **核心能力 — 目标拆解思考**：
  - AI 必须先理解用户的最终目标，而非机械拆分文字
  - 分析目标之间的依赖关系和先后顺序（例如"学完课程"必须在"通过考试"之前）
  - 评估每个目标的难度和所需时间，合理分配到不同的周
  - 判断推进节奏：前期密集还是均匀分布，是否需要留出缓冲周
  - 结合用户的行为风格（精力高峰、擅长的工作方式、历史完成率）来调整方案
  - 确保月底回看时，目标是可以达成的
- **风格要求**：
  - 格式必须与用户习惯完全一致
  - 使用用户惯用的语言风格和措辞
  - 尊重用户的时间偏好
  - 保留用户的固定日常事项
  - 任务量与用户习惯匹配

#### Prompt 3：修改学习（Modification Learning）

- **触发时机**：用户修改了 AI 生成的计划并保存后
- **输入**：原始计划 + 修改后计划 + 当前风格画像
- **输出**：修改分析 + 洞察 + 风格画像更新建议（JSON）
- **关键要求**：
  - 识别增删改的具体内容
  - 区分"偶然调整"和"系统性偏好"
  - 输出可直接应用的画像更新字段

#### Prompt 4：今日总结（Daily Summary）

- **触发时机**：用户触发"今日总结"按钮
- **输入**：今日计划 + 完成状态 + 用户修改记录 + 近期完成趋势
- **输出**：Markdown 格式的总结报告
- **关键要求**：
  - 包含完成率统计
  - 给出正向反馈
  - 分析未完成原因
  - 列出顺延事项

#### Prompt 5：明日计划（Tomorrow's Plan）

- **触发时机**：今日总结完成后自动触发
- **输入**：风格画像 + 本周剩余任务 + 今日未完成项 + 今日修改记录 + 历史示例
- **输出**：符合用户风格的明日计划
- **关键要求**：
  - 风格与用户一致，不添加额外标注
  - 未完成的重要事项自然融入明日计划
  - 体现从修改中学到的偏好
  - 根据近期完成率调整工作量

### 4.2 AI 模型策略

通过 OpenRouter API 统一调用，用户自带 API Key。

| 角色 | 推荐模型 | 用途 |
|------|---------|------|
| 主力模型 | `google/gemini-3-flash-preview` | 日常计划生成、风格分析、总结（快速、便宜、中文好） |
| 图片解析备选 | `google/gemini-3-pro-image-preview` | 手写计划照片、复杂排版截图的高精度识别 |
| 深度分析备选 | `openai/gpt-5.2-chat` | 周/月深度总结分析、复杂拖延模式识别 |

默认使用 `gemini-3-flash-preview`，用户可在设置中切换。

### 4.3 API 调用方式

```javascript
// OpenRouter API 调用
fetch('https://openrouter.ai/api/v1/chat/completions', {
    method: 'POST',
    headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json',
        'HTTP-Referer': window.location.href,
        'X-Title': 'AI Plan Assistant'
    },
    body: JSON.stringify({
        model: selectedModel,
        messages: [
            { role: 'system', content: systemPrompt },
            { role: 'user', content: userInput }
        ]
    })
});
```

## 五、界面设计

### 5.1 整体架构

单页面 Web 应用，Tab 标签导航，四个主要页面：

| Tab | 页面 | 核心功能 |
|-----|------|---------|
| 📎 上传 | 风格训练 | 上传历史计划，分析风格 |
| 📋 计划 | 主面板 | 月目标输入，周/日计划展示，勾选完成，编辑修改 |
| 📊 分析 | 总结分析 | 今日总结报告 + 明日计划预览 |
| ⚙️ 设置 | 配置 | API Key、模型选择、数据管理 |

### 5.2 设计风格

- 深紫色/靛蓝色主色调（与现有工具风格一致）
- 白色卡片 + 柔和阴影
- 绿色表示成功/完成状态
- 响应式布局，支持不同屏幕尺寸
- 中文界面

### 5.3 计划面板交互

- AI 生成的计划以自然格式展示
- 以 `- ` 或 `* ` 开头的行识别为任务项，渲染为可勾选复选框
- `## ` 开头的行识别为分组标题
- 用户点击"编辑"进入文本编辑模式，保存后记录修改差异
- 系统同时保存原始版本和修改版本

## 七、AI 学习闭环（核心机制）

这是本产品区别于普通 AI 工具的核心竞争力：

```
第1天：
  用户上传历史计划 → AI 分析 → 风格画像 v1（初始版本）
  用户输入月目标 → AI 按风格生成日计划
  用户修改计划 → AI 检测差异 → 风格画像 v1.1
  用户勾选完成 → AI 总结 → 预生成明日计划

第2天：
  用户查看明日计划 → 接受或修改
  如有修改 → AI 继续学习 → 风格画像 v1.2
  一天结束 → AI 总结 → 明日计划（更精准）

第N天：
  风格画像 vN → AI 越来越懂用户 → 修改越来越少
```

**风格画像是活的数据**，它不是一次性生成的，而是随着每次用户交互持续进化。

## 十、设计原则

1. **用户零 Prompt 负担**：所有 Prompt 内置在系统中，用户不需要写任何提示词
2. **目标导向的智能规划**：AI 不是简单的文字拆分工具，而是具备思考能力的规划参谋——理解目标本质、分析依赖关系、规划推进节奏、结合用户行为风格，帮助用户真正达成目标
3. **风格一致性优先**：AI 生成的计划必须"看起来像用户自己写的"
4. **最小交互**：用户只需要输入月目标 + 接受/拒绝计划，日常操作极简
5. **持续进化**：每一次用户修改都是 AI 学习的机会
6. **渐进式复杂度**：MVP 先验证核心价值，再逐步添加后端和发布能力
